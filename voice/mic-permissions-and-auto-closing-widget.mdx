---
title: "Mic Permissions and Auto-Closing Widget"
---

Voice conversations require microphone access. If the browser blocks the mic or the voice widget closes unexpectedly, follow the steps below.

#### **Allowing microphone access**

1. Open the AI Double profile or embedded chatbot
2. Click the voice/call button
3. When the browser prompts for microphone permission, click Allow

#### If the prompt did not appear or was blocked

1. Click the lock or info icon in the browser address bar
2. Find Microphone under Site Settings
3. Set it to Allow
4. Refresh the page and try again

#### **Browser-specific steps**

- Chrome - Address bar → Lock icon → Site settings → Microphone → Allow
- Safari - System Preferences → Security & Privacy → Microphone → allow browser
- Firefox - Address bar → Lock icon → Permissions → Use the Microphone → Allow

#### **If the voice widget closes automatically**

- This can happen when mic access is blocked - Grant permission and reload
- Check that no other app is using the microphone exclusively
- On mobile, ensure the browser has microphone permission in system settings
- Try a different browser if the issue persists

#### **What to know**

- HTTPS required - Voice only works on secure (https://) connections
- Mobile browsers may require permission from system settings, not just the browser
